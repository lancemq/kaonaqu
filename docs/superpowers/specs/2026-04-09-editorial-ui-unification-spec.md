# 2026-04-09 全站 Editorial UI 统一规范

## 目标

为 `首页 / 新闻频道 / 学校频道 / 新闻专题页 / 学校专题页 / 新闻详情页 / 学校详情页` 建立一套统一的展示系统，让用户在不同频道之间切换时，始终感到这是同一个产品，而不是一组风格接近但结构割裂的页面。

统一方向不是“全站一个皮肤”，而是统一：

- 信息层级
- 页面骨架
- 卡片系统
- 文本层级
- 交互反馈
- 频道间的阅读节奏

---

## 视觉方向

### 总体气质

采用“上海升学编辑部”风格：

- 米白纸面背景
- 黑色信息条与重点卡
- 橙色作为重点提示色
- 细灰边线与浅投影
- 标题有媒体头版感，正文有资料库可信感

### 字体策略

- 标题：`Newsreader`
- 正文：`Geist`
- 标签 / 时间 / 辅助元信息：`JetBrains Mono`

### 色彩规范

- 页面背景：`#F2F3F0`
- 深一层背景：`#E7E8E5`
- 纸面卡片：`#FFFFFF`
- 轻强调底：`#E9E3D8`
- 正文主色：`#191919`
- 辅助文字：`#666666`
- 边框线：`#CBCCC9`
- 强调色：`#FF8400`
- 强调深色：`#804200`
- 深色卡片：`#111111`
- 深色卡片文字：`#F2F3F0`

### 视觉原则

- 黑底只用于重点提醒、频道状态、编辑推荐，不大面积滥用
- 橙色只用于栏目锚点、时间标签、重点提示，不用来铺大背景
- 每个页面至少有一块强重点区域，但不超过两块
- 标题与正文的对比必须明显，避免全页都是一样的视觉强度

---

## 设计系统

### 基础模块

全站只保留 6 类基础模块：

1. `Channel Hero`
2. `Status Bar`
3. `Section Head`
4. `Surface Card`
5. `Side Rail`
6. `Footer Strip`

### Channel Hero

适用页面：

- 首页
- 新闻频道
- 学校频道
- 新闻专题首页
- 学校专题首页
- 新闻详情页
- 学校详情页

统一结构：

- 左侧：频道名 / 主标题 / 一句话导语 / 入口按钮或 tags
- 右侧：本周最值得先看 / 快速判断卡 / 频道辅助信息

Hero 不承担所有内容，只承担“告诉用户现在这页是什么、先看哪里”。

### Status Bar

统一为黑底横向条：

- 展示频道状态
- 展示数量信息
- 展示更新频率或推荐阅读顺序

只放 2 到 4 组信息，不塞过多说明。

### Section Head

统一结构：

- 左：`overview-label + 主标题`
- 右：一个轻操作入口，例如“查看全部 / 专题总览 / 完整时间线”

不允许页面内每个 section head 都换结构。

### Surface Card

统一卡片系统分三种：

- `Surface / Default`
  用于普通信息卡、正文卡、列表卡、结果卡
- `Surface / Warm`
  用于强调但不需要深色压强的卡
- `Surface / Dark`
  用于编辑推荐、重点提醒、状态解释

交互统一：

- hover：轻微上浮
- 阴影略增强
- 边框色轻微加强

### Side Rail

右侧侧栏统一做 3 到 4 块：

- 阅读顺序
- 关联专题
- 快速入口
- 同区 / 同类 / 相关阅读

不再允许每个详情页临时发明侧栏结构。

### Footer Strip

统一两段式：

- 左：页面定位
- 右：内容范围或数量摘要

## 页面级规范

## 1. 首页

### 页面角色

首页是“总编排首页”，不是单纯入口页。

### 推荐结构

1. 顶部导航
2. 首页 Hero
3. 今日 / 本周关键内容区
4. 新闻主卡区
5. 深色重点判断区
6. 时间线区
7. 学校入口区
8. 16 区入口区
9. 继续阅读 CTA

### 首页文案原则

- 不解释网站在做什么
- 直接告诉用户“先看什么”
- 用“判断”和“下一步”做引导

### 首页视觉重点

- 首页必须有一块黑底编辑推荐区
- 首页的右侧卡必须承担“本周最值得先看”
- 首页中部应形成“新闻 / 时间 / 学校”三段节奏

---

## 2. 新闻频道页

### 页面角色

新闻频道是“新闻编辑台”，不是普通列表页。

### 推荐结构

1. Hero：年度新闻定位 + 推荐主新闻
2. Status Bar：新闻数量 / 政策数量 / 学校动态数
3. 本周重点 3 卡
4. 新闻筛选与分页列表
5. 右侧阅读顺序与专题入口
6. 频道页脚

### 统一规则

- 第二条新闻卡保留深色推荐位
- 列表卡统一成一种结构
- 筛选器统一成胶囊样式
- 分页统一成按钮 + 状态 chip

### 侧栏内容固定顺序

1. 阅读顺序
2. 高频政策问答
3. 政策概念速查
4. 时间线
5. 关联学校或专题推荐

---

## 3. 学校频道页

### 页面角色

学校频道是“学校检索台”。

### 推荐结构

1. Hero：数据库定位 + 搜索框 + 数据摘要
2. Status Bar：学校总量 / 区域覆盖 / 最近更新时间
3. 左侧：筛选控制台
4. 右侧：结果区
5. 结果统计卡
6. 学校结果卡列表
7. 分页

### 统一规则

- 左侧控制台固定为“当前条件 / 控制台 / 特征筛选 / 快速入口 / 数据口径”
- 结果卡统一结构：
  - 第一行：学校名 + 区域 / 学段 / 办学性质
  - 第二行：一句学校判断
  - 第三行：标签与进入详情
- 标签与培养方向统一走 chip 系统

---

## 4. 新闻专题页

适用：

- `zhongkao-special`
- `gaokao-special`
- `admission-timeline`
- `policy-glossary`
- `policy-faq`
- `policy-deep-dive`

### 页面角色

专题页是“问题解决页”，不是普通内容归档页。

### 统一模板

1. Hero：专题标题 + 导语 + 快速入口
2. Status Bar：适合谁看 / 先看什么 / 更新时间
3. 主内容区
4. 方法卡 / FAQ / 概念关系 / 时间线 / 文件阅读顺序
5. 右侧阅读顺序与关联专题

### 统一要求

- 每个专题页首屏都必须有“先看什么”
- 每个专题页都要有“继续阅读”入口
- FAQ、概念卡、流程卡、时间线卡属于同一视觉家族

---

## 5. 新闻详情页

### 页面角色

新闻详情页是“正文 + 决策辅助”页。

### 统一结构

1. Hero：标题 / 日期 / 分类 / 摘要
2. 正文主栏
3. 右侧：
   - 关键信息
   - 相关阅读
   - 关联学校
   - 继续阅读专题
4. 底部：返回频道 / 进入专题

### 正文规则

- 正文样式克制
- 标题层级统一
- 列表、引用、说明块使用同一套正文格式
- 右侧侧栏宽度固定

---

## 6. 学校详情页

### 页面角色

学校详情页是“学校档案页”。

### 统一结构

1. Hero：学校名 / 一句话画像 / 快速标签
2. Stats：区域 / 学段 / 办学属性 / 热度
3. 主栏：
   - 学校概览
   - 正文判断
4. 侧栏：
   - 学校速览
   - 联系与服务信息
   - 同区学校
   - 培养与检索
5. Footer

### 规则

- 强调“档案感”和可信赖感
- 不要做成新闻详情页换皮
- 详情页里“学校定位”必须是深色重点卡

## 组件映射建议

为了真正统一，建议后续把当前页面收敛为以下组件：

- `SiteHeader`
- `ChannelHero`
- `ChannelStatusBar`
- `SectionHead`
- `EditorialCard`
- `EditorialDarkCard`
- `FilterChips`
- `SideRail`
- `SideRailCard`
- `RichArticle`
- `StatsGrid`
- `Pager`
- `FooterStrip`

其中：

- 首页、新闻页、学校页共享 `ChannelHero / SectionHead / FooterStrip`
- 新闻专题页和新闻详情页共享 `RichArticle / SideRail / StatsGrid`
- 学校频道和学校详情页共享 `StatsGrid / EditorialCard / SideRail`

## 文案与内容密度规则

### 标题

- 标题只说用户关心的结果
- 不用“平台 / 入口 / 汇总 / 频道导航”这类产品化措辞做主标题

### 摘要

- 首页卡片摘要：1 到 2 行
- 新闻卡摘要：2 行以内
- 学校卡摘要：一句可判断信息
- 详情页摘要：一段，不超过 90 字

### 节奏

- 每一屏只能有一个绝对重点
- 每一块区内容量要控制，不要过度堆砌
- 用户先扫，再细读，再比较

---

## 开发落地顺序

### 第一阶段

- 固化这份规范
- 用统一组件替换首页、新闻频道、学校频道顶部骨架

### 第二阶段

- 统一新闻专题页模板
- 统一新闻详情页正文模板

### 第三阶段

- 统一学校详情页
- 统一区县页与学校比较页

### 第四阶段

- 清理遗留样式
- 压缩样式分支
- 统一移动端响应式节奏

---

## 本轮实施建议

如果按收益排序，优先改这 4 个页面：

1. 首页
2. 新闻频道页
3. 学校频道页
4. 学校详情页

这 4 个页面统一后，整站感知会先稳定下来；再把新闻专题页和新闻详情页接入统一模板，用户体验会明显连贯。
