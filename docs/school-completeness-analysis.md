# 学校数据完整度分析

> 生成时间：2026-07-02 ｜ 数据源：`data/schools.json`（883 所）+ `content/schools/*.md`

## 一、分析背景

`profileDepth` 字段已统一升级到 `enhanced`/`priority`（`scripts/check-school-enrichment.mjs` 强制校验），仅代表"已过基础补全"。本分析按 **md 详情深度 + 联系招生字段完整度** 重新分档，识别真正需要补全的学校。

## 二、分档定义

| 档位 | 标准 | 含义 |
|------|------|------|
| **A 精品完整** | md≥1200字 + 章节≥10 + 联系招生 6/6 齐全 | 信息完整、详情丰富 |
| **B 标准完整** | md≥800字 + 章节≥8 + 联系招生≥5/6 | 信息较完整，少量字段缺失 |
| **C1 待补全** | md<800字 或 章节<8 或 缺≥2 项联系信息 | 需补 md 详情或字段 |
| C2/D | — | 当前为 0 |

**联系招生 6 项**：`phone` `admissionCode` `address` `website` `admissionNotes` `admissionMethods`

**md 深度章节**：`学校概览` `基础信息` `关注重点` `招生与路径` `培养方向` `适合谁` `阅读提示` `公开信息入口` `联系与地址` `官方对口查询`

## 五、分批补全方案

| 批次 | 优先级 | 范围 | 数量 | 工作内容 |
|------|--------|------|------|----------|
| **批次 1** | 高 | md<800字 + 同时缺 phone+admissionCode | 126 所 | 补 md 详情（加深"关注重点/培养方向"章节）+ 查 phone/admissionCode |
| **批次 2** | 中 | md<800字、字段齐全 | 46 所 | 仅补 md 详情内容 |
| **批次 3** | 中 | md≥800字、缺 phone 或 admissionCode（C1） | 314 所 | 仅查 phone/admissionCode |
| **批次 4** | 低 | B 档缺 phone 或 admissionCode | 269 所 | 查字段，升 A 档 |


## 七、补全操作指引

### 单批补全（推荐）

```bash
# 1. 导出某批次名单为限制文件（示例：批次 1）
#    手工编辑 scripts/enrich-school-markdown-unified.mjs 的输入或加 --filter 参数

# 2. 预演（限制数量）
npm run data:schools:enrich:dry -- --limit 20

# 3. 正式补全
npm run data:schools:enrich

# 4. 校验
npm run data:validate
```

### 字段补全（phone/admissionCode）

phone 和 admissionCode 缺失量大（580 所缺 phone、609 所缺 admissionCode），建议：

1. 优先从各区教育局/学校官网补 `phone`（`address`/`website` 已全，可从官网联系页提取）
2. `admissionCode` 为中招志愿编码，初中很多不参与高中招生编码，可对初中阶段学校标记为"不适用"而非缺失
3. 高中阶段学校 admissionCode 应从 `shmeea.edu.cn` 招生计划表补全

### md 详情补全

参考 A 档学校的 md 结构（13 节，1200+ 字），重点加深：
- **关注重点**：办学特色、近年变化、家长关注点
- **培养方向**：升学路径、特色项目、出口方向
- **适合谁**：适配家庭画像
