# 考哪去（kaonaqu）系统详细设计

> 面向上海学生与家长的升学信息站。本文档基于仓库实际代码（`shared/*`、`lib/*`、`crawler/*`、`data/*`、`app/*`）梳理系统架构、功能模块与数据模型，并给出**数据库表设计规范**（含现状映射与规范化建议）。
>
> 文档版本：2026-07-08 · 阅读对象：开发 / 架构 / 运维

---

## 1. 项目概述

### 1.1 定位与目标
- **定位**：聚合上海中考 / 高考新闻政策、学校信息、区域专题与初高中知识体系的本地优先（local-first）信息站。
- **目标用户**：上海小升初 / 初升高 / 高考生及家长。
- **核心价值**：把分散在教委、考试院、学校官网、社区与社媒的升学信息，归一化后统一检索与对比。

### 1.2 技术栈
| 层 | 技术 |
|---|---|
| 框架 | Next.js 16（App Router），`revalidate` 默认 86400（ISR 一天） |
| 视图 | React 19（多数页面为 Server Component + 少量 Client Component） |
| 样式 | 原生 CSS（`styles/index.css` → `base.css` → `theme-*.css`），设计令牌集中在 `:root` |
| 数据 | 本地 `data/*.json` + `content/*.md`，可选 Supabase（Postgres）同步 |
| 采集 | Node 爬虫（`crawler/src/*`，cheerio + axios），由 Vercel Cron 每日触发 |
| 部署 | Vercel（Root = 仓库根，Build = `next build`，Cron 需 `CRON_SECRET`） |
| 语言 | JS 为主；`strict:false` 的 TS 仅用于少数引擎文件（score-match / simulator） |

### 1.3 当前数据规模（仓库实测）
- 学校 `schools.json`：**911 条**（含 `priority` / `foundation` 两种 profileDepth）
- 新闻 `news.json`：**207 条**
- 政策 `policies.json`：**15 条**
- 区域 `districts.json`：**16 个区**（由 `DISTRICT_CATALOG` 派生）
- 知识库 `knowledge-pages.json`：按学科 / 年级组织的 legacy 导出（约数十个学科页）

## 2. 系统总体架构

### 2.1 分层架构图

```
┌──────────────────────────────────────────────────────────────┐
│                       用户（浏览器）                          │
└───────────────┬──────────────────────────────┬──────────────┘
                │ HTTP                         │ HTTP (Cron)
                ▼                              ▼
┌───────────────────────────┐      ┌──────────────────────────┐
│   Next.js App (App Router) │      │  Vercel Cron (每日 17:00) │
│  app/ 页面 + app/api/*      │      │  /api/cron-refresh        │
└──────────┬────────────────┘      └────────────┬─────────────┘
           │                                     │
           │ 读取/渲染                            │ 触发
           ▼                                     ▼
┌──────────────────────┐              ┌─────────────────────────┐
│  lib/*  (页面数据组织) │              │  crawler/* 采集流水线     │
│  shared/content-service│             │  raw → processed → data  │
│  shared/data-store    │             │  mergeRecords 合并       │
└──────────┬───────────┘              └────────────┬────────────┘
           │ 加载                                   │ 写回
           ▼                                        ▼
┌──────────────────────────────────────────────────────────────┐
│              权威运行数据源（data/ + content/）                 │
│  districts.json schools.json policies.json news.json          │
│  knowledge-pages.json  +  content/*.md（长文详情）              │
└──────────────────────────────┬───────────────────────────────┘
                               │ 可选同步（KNQ_SUPABASE_*）
                               ▼
                     ┌──────────────────────┐
                     │  Supabase (Postgres) │  schools / school_contents
                     └──────────────────────┘
```

### 2.2 数据流原则（local-first）
```
content/*.md + data/*.json  →  lib/* 与 shared/*  →  app/* 页面 / app/api/*
                                                   →  crawler/*  →  data/
```
- **`data/` 是唯一权威运行数据源**：`districts / schools / policies / news`（+ `knowledge-pages`）。
- **`content/` 存 Markdown 长文**：学校详情、政策解读、新闻正文，由 `*.contentFile` 字段关联。
- **`shared/data-store.js`** 通过 `KAONAQU_RUNTIME_ROOT_DATA_DIR` 定位数据目录；找不到本地文件时回退到 `require('../data/*.json')` 的打包快照（serverless 友好）。
- **写入**用 `writeLocalJson`（2 空格 JSON + 尾换行）；记录合并走 `mergeRecords` / `pickPreferredRecord`，按 `updatedAt` 时间 + 字段完整度择优。

### 2.3 部署与定时
- `vercel.json`：负责 `cleanUrls` + 每天触发 `/api/cron-refresh`。
- `/api/cron-refresh` 需 `Authorization: Bearer $CRON_SECRET`；在 `/tmp` 下跑 crawler 并 `mergeDataStore` 替换本地数据。
- 当前 Supabase / Blob 同步默认关闭（`isSupabaseConfigured()` 为 false 时降级读本地文件）。

## 3. 功能模块设计

### 3.1 首页（`/`）
- Server Component，`data-page="home"`，聚合四频道入口与最新内容。

### 3.2 新闻频道（`/news` 及子路由）
- `news-page-client.js` 驱动交互。
- 列表支持：考试类型（中考/高考）、新闻类型（招生/学校/考试）、分类、来源类型、关键词检索。
- 详情 `/news/[id]` 渲染 Markdown 正文。
- **专题页**（独立路由，体现内容深度）：
  - `/news/admission-timeline` 招生时间线
  - `/news/zhongkao-special` / `/news/gaokao-special` 中/高考专题
  - `/news/policy-faq` 政策问答、`/news/policy-glossary` 政策术语表
  - `/news/sports-reform` 体育改革专题

### 3.3 学校频道（`/schools` 及子路由）
- 主列表 `schools-page-client.js`：按区、学段（初中/高中/完全中学）、办学性质（公办/民办）、关键词筛选，按优先级排序（`eliteCohort` 非空优先 + `schoolKeyLevel` 重点优先）。
- **学校详情 `/schools/[id]`**：简介、录取说明、特色、分数线、对口/去向、丰富档案（history/programs/competitions/notablePeople）。
- **对比 `/schools/compare`**：`compare-bag` 收藏篮 + `schools-compare-client.js` 多校并排对比。
- **估分择校 `/schools/score-match`**：`score-match-engine.ts`，输入「分数 + 区域 + 考试类型」，按 tier 参考区间给出 **冲刺 / 匹配 / 保底** 三档高中建议。
  - 中考：覆盖全部高中，用 tier 参考区间（`source='tier_reference'`）。
  - 高考：仅对有 rich profile scoreLines 的头部校做精确匹配（`source='rich_profile'`）。
- **模拟器 `/schools/simulator`**：`schools-simulator-client.tsx`，志愿/录取模拟。
- **群组 `/schools/groups`**：按 `group` 字段聚合（如「上中系」「华二系」）。
- **区域 `/schools/district/[district]`**：按区专题。

### 3.4 知识体系频道（`/knowledge` 与 `/knowledge/[[...slug]]`）
- 按学段 / 年级 / 学科整理学习内容。
- `knowledge-page.js` 渲染 `knowledge-pages.json` 中的 `richBlocks`（嵌套 React 式 JSON：element/text 节点）。
- 数据来自 legacy 导出快照，当前为静态内容页。

## 4. 关键设计与约定

### 4.1 API：单一 catch-all 入口
- 所有 REST 走 `app/api/[...slug]/route.js` → `shared/api-router.js` 的 `handleApiRequest({method, pathname, query, body})`。
- 业务在 `shared/content-service.js`（CRUDL + 搜索 + 归一化），schema 在 `shared/data-schema.js`。
- **新增接口只需在 `api-router.js` 加 `pathname` 分支**，无需新建 route 文件。
- 端点：`/api/districts`、`/api/schools`(CRUDL)、`/api/policies`(CRUDL)、`/api/news`(CRUDL)、`/api/search?q=`。

### 4.2 `lib/` 与 `shared/` 边界
- `lib/`：内容解析与页面数据组织（多 `.mjs`，供页面 server 端 import）。
- `shared/`：API 路由、schema、数据读写与服务层（CommonJS，被 API route 与 cron 通过 `createRequire` 桥接）。
- `app/` 下 server 文件用 ESM import，需要 `shared/` 时通过 `createRequire(import.meta.url)`。

### 4.3 样式令牌系统
- `styles/index.css` 依次 import `base.css` → `theme-{home,news,schools,knowledge}.css`。
- **设计令牌统一定义在 `base.css` 的 `:root`**（`--site-*`、`--font-*`、`--channel-*`）；各频道 `theme-*.css` 只覆盖差异变量，禁止重复声明整套令牌。
- 频道切换靠 `body[data-page="..."]` 选择器（由 `components/body-page-flag.js` 设置）。

### 4.4 数据归一化与合并（核心算法）
- `normalizeSchool / normalizePolicy / normalizeNews`：清洗字符串、推断来源类型、推断考试/新闻分类、生成 slug、`buildSchoolTags` 自动打标签。
- `mergeRecords(...collections)`：按 `id` 聚合，对同 id 记录用 `pickPreferredRecord`（时间新者优先，时间相同比完整度）做字段级 `mergeRecordPair` 合并。
- `DISTRICT_CATALOG` 为硬编码 16 区目录；`SOURCE_TYPE_MAP` 将来源名归一为 `official/third_party/community/social/unknown` 并赋 `confidence`。

## 5. 采集与数据处理流水线

### 5.1 数据源（`crawler/src/sources/registry.json`）
| 类型 | 代表源 |
|---|---|
| policy（official） | 上海市教委中招政策栏目 `edu.sh.gov.cn` |
| school（official） | 教委实验性示范性高中名单、特色普通高中名单 |
| news（official） | 上海市教育考试院消息速递 `shmeea.edu.cn` |
| news/school/policy（social/community） | 家长帮、小红书、微信公众号、学而思社区等 |

### 5.2 采集器（`crawler/src/crawlers/*`）
`news-official`、`policies-official`、`schools-official`、`schools-websites`、`social-platforms`、`education-forums`、`community-fallback`、`school-enrichment`。

### 5.3 ETL（`crawler/src/process-data.js`）
```
raw/(*.json)  →  normalize(shared/data-schema)  →  processed/  →  data/*.json
```
`RAW_OUTPUT_FILES` 包括 `official-news / official-policies / official-schools / social-news / social-policies / social-schools / forum-schools / junior-schools-tier / admission-discussions`。

### 5.4 维护脚本（package.json）
- 内容生成：`data:news:markdown`、`data:policies:markdown`、`data:schools:markdown`
- 学校补全：`data:schools:enrich(:dry)`、`data:schools:signals`
- 校验：`data:validate`（链接校验 + 补全度校验 + 结构校验）
- SEO：`seo:build`（sitemap.xml + baidu_urls.txt）
- 同步：`data:sync:supabase` / `data:sync:from-supabase`

> ⚠️ **现状缺口**：`scripts/` 目录当前为空，但 `package.json` 中大量脚本（如 `generate-seo.js`、`sync-detail-markdown.mjs`、`enrich-school-markdown-unified.mjs`、`migrate-schools-to-supabase.js`）指向该目录；`supabase/migrations/` 亦为空。**若需恢复 Supabase 同步与内容脚本，应先补回 `scripts/` 并补建迁移 SQL。**

## 7. 数据库表设计（核心）

### 7.1 设计原则
1. **Supabase = Postgres**：使用 `text[]`、`jsonb`、`timestamptz`、外键与部分索引。
2. **与代码映射一致**：表列名采用 `rowToSchool` 中已用的 snake_case（`slug / district_name / school_stage_label / admission_info ...`），降低迁移成本。
3. **来源归一化**：抽取 `sources` 表，避免每行长串内嵌来源对象。
4. **关系显式化**：学校↔学校（对口去向）、新闻↔学校（关联）拆成关联表。
5. **可演进**：分数线 / 丰富档案先以独立表 + `jsonb` 承载，后续可进一步规范化。

### 7.2 现状：Supabase 已映射表（代码中已调用的列）
`shared/data-store.js` 的 `rowToSchool` 与 `shared/supabase-client.js` 已隐含两张表：

**`schools` 表（已映射列）**
| 列 | 类型 | 说明 |
|---|---|---|
| `id` | bigint PK | 数值主键（dbId） |
| `slug` | text UNIQUE | 应用层 id（如 `xuhui-上海中学`），关联 `contentFile` |
| `name` | text | 校名 |
| `district_name` | text | 区名（FK 语义 → districts） |
| `school_stage_label` | text | 完全中学/高中/初中 |
| `school_property_label` | text | 公办/民办/… |
| `tier` | text | 梯队（旧字段，兼容） |
| `school_key_level` | text | 市重点/三公/区重点… |
| `elite_cohort` | text | 四校/八大…（排序用） |
| `group` | text | 教育集团（上中系…） |
| `address` / `phone` / `website` | text | 联系信息 |
| `founding_year` | int | 建校年 |
| `is_boarding` / `is_international` | bool | 寄宿/国际化 |
| `image` | text | 图片路径 |
| `description` / `achievements` | text | 简介/成绩 |
| `admission_notes` | text | 录取说明 |
| `admission_info` | jsonb | `{code, methods[], routes[]}` |
| `profile_depth` | text | priority/foundation |
| `features` | text[] | 特色标签 |
| `content_file` | text | Markdown 路径 |
| `created_at` / `updated_at` | timestamptz | 审计 |

**`school_contents` 表（已被引用，建议结构）**：存放学校长文详情，避免与结构化字段混放。
| 列 | 类型 | 说明 |
|---|---|---|
| `school_slug` | text PK / FK→schools(slug) | 关联学校 |
| `markdown` | text | 详情正文 |
| `updated_at` | timestamptz | |

### 7.3 建议规范化表设计（建表 SQL）
> 以下为**完整支撑现有功能**的目标 schema（PostgreSQL / Supabase）。可直接作为 `supabase/migrations/0001_init.sql` 落地。

```sql
-- ============ 基础维度 ============
CREATE TABLE districts (
  id          text PRIMARY KEY,                 -- huangpu / xuhui ...
  name        text NOT NULL,                    -- 黄浦区
  description text
);

CREATE TABLE sources (
  id          text PRIMARY KEY,                 -- 归一后来源标识（如 slugify(name)）
  name        text NOT NULL,
  type        text NOT NULL                     -- official/third_party/community/social/unknown
                CHECK (type IN ('official','third_party','community','social','unknown')),
  url         text,
  confidence  numeric(3,2) CHECK (confidence BETWEEN 0 AND 1)
);

-- ============ 学校主表 ============
CREATE TABLE schools (
  id                bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  slug              text UNIQUE NOT NULL,       -- 应用层 id
  name              text NOT NULL,
  district_id       text REFERENCES districts(id),
  district_name     text,
  school_stage_label text,                      -- 完全中学/高中/初中
  school_property_label text,                   -- 公办/民办/...
  tier              text,
  school_key_level  text,
  elite_cohort      text,
  group_name        text,                       -- 教育集团（原 group）
  address           text,
  phone             text,
  website           text,
  founding_year     int,
  is_boarding       boolean DEFAULT false,
  is_international  boolean DEFAULT false,
  image             text,
  description       text,
  achievements      text,
  admission_notes   text,
  admission_info    jsonb,                       -- {code, methods[], routes[]}
  profile_depth     text CHECK (profile_depth IN ('priority','foundation')),
  features          text[],
  content_file      text,
  created_at        timestamptz DEFAULT now(),
  updated_at        timestamptz DEFAULT now()
);

-- 学校长文详情
CREATE TABLE school_contents (
  school_slug   text PRIMARY KEY REFERENCES schools(slug) ON DELETE CASCADE,
  markdown      text,
  updated_at    timestamptz DEFAULT now()
);

-- 分数线（原 rich profile 的 scoreLines）
CREATE TABLE school_score_lines (
  id           bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  school_slug  text NOT NULL REFERENCES schools(slug) ON DELETE CASCADE,
  year         text,                            -- 2021..2025
  batch        text,                            -- 1-15 志愿统一招生
  score        numeric,                         -- 705
  scope        text,                            -- 委属/市重
  source_label text,
  source_url   text,
  source_note  text
);

-- 对口/录取去向（原 admissionRoutes：初中→高中）
CREATE TABLE school_admission_routes (
  id              bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  from_school_slug text NOT NULL REFERENCES schools(slug) ON DELETE CASCADE, -- 初中
  to_school_slug   text NOT NULL REFERENCES schools(slug) ON DELETE CASCADE, -- 高中
  route_type       text,                        -- 中考对口
  year             int,
  count            int,
  confidence       text CHECK (confidence IN ('high','medium','low'))
);

-- 丰富档案（history/programs/competitions/notablePeople）
CREATE TABLE school_rich_profiles (
  school_slug  text PRIMARY KEY REFERENCES schools(slug) ON DELETE CASCADE,
  badge        text,
  image        jsonb,
  history      jsonb,
  programs     jsonb,
  competitions jsonb,
  notable_people jsonb,
  updated_at   timestamptz DEFAULT now()
);

-- ============ 政策 ============
CREATE TABLE policies (
  id               text PRIMARY KEY,            -- slug
  title            text NOT NULL,
  category         text,                        -- 中招政策
  year             int,
  district_id      text REFERENCES districts(id),
  district_name    text,
  summary          text,
  content_file     text,
  source_id        text REFERENCES sources(id),
  tags             text[],
  target_audience  text,
  importance       text CHECK (importance IN ('high','medium','low')),
  published_at     timestamptz,
  updated_at       timestamptz DEFAULT now()
);

-- ============ 新闻 ============
CREATE TABLE news (
  id                  text PRIMARY KEY,          -- slug
  title               text NOT NULL,
  news_type           text CHECK (news_type IN ('admission','school','exam')),
  exam_type           text CHECK (exam_type IN ('gaokao','zhongkao','')),
  category            text,
  summary             text,
  content             text,
  content_md          text,
  content_file        text,
  published_at        timestamptz,
  source_id           text REFERENCES sources(id),
  primary_school_id   text,
  school_link_reason  text,
  school_link_confidence numeric(3,2),
  updated_at          timestamptz DEFAULT now()
);

-- 新闻↔学校多对多关联（原 relatedSchoolIds）
CREATE TABLE news_school_links (
  news_id      text NOT NULL REFERENCES news(id) ON DELETE CASCADE,
  school_slug  text NOT NULL REFERENCES schools(slug) ON DELETE CASCADE,
  is_primary   boolean DEFAULT false,
  PRIMARY KEY (news_id, school_slug)
);

-- ============ 知识体系（legacy 导出，先 jsonb 承载） ============
CREATE TABLE knowledge_pages (
  slug         text PRIMARY KEY,                -- chemistry / chemistry-grade8
  grade        text,                            -- grade8 / senior1 ...
  title        text,
  description  text,
  breadcrumb   jsonb,
  rich_blocks  jsonb,                           -- 嵌套 React 式节点
  updated_at   timestamptz DEFAULT now()
);

-- ============ 采集审计 ============
CREATE TABLE crawl_sources (
  id           text PRIMARY KEY,                -- registry.json 中的 source id
  name         text,
  type         text,
  authority    text,
  host         text,
  entry_url    text,
  frequency    text,
  notes        text
);

CREATE TABLE crawl_runs (
  id            bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  source_id     text REFERENCES crawl_sources(id),
  started_at    timestamptz DEFAULT now(),
  finished_at   timestamptz,
  status        text CHECK (status IN ('running','success','failed')),
  records_fetched int,
  records_merged  int
);
```

### 7.4 索引、约束与枚举建议
| 表 | 索引 / 约束 | 目的 |
|---|---|---|
| schools | `idx_schools_district(district_id)`、`idx_schools_stage(school_stage_label)`、`idx_schools_key_level(school_key_level)`、`GIN idx_schools_features(features)` | 列表筛选 / 全文检索加速 |
| schools | 部分索引 `WHERE elite_cohort IS NOT NULL` | 头部校置顶排序 |
| news | `idx_news_published(published_at DESC)`、`idx_news_exam(exam_type)`、`GIN idx_news_title_gin(title)` | 时间线、考试筛选 |
| policies | `idx_policies_year_district(year, district_id)` | 政策检索 |
| school_score_lines | 唯一 `(school_slug, year, batch)` | 防重复分数线 |
| school_admission_routes | `(from_school_slug)`、`(to_school_slug)` | 对口/去向双向查 |
| news_school_links | 反向 `(school_slug, news_id)` | 学校相关新闻 |

**枚举（CHECK）**：
- `source.type`：`official / third_party / community / social / unknown`
- `news.exam_type`：`gaokao / zhongkao / ''`
- `news.news_type`：`admission / school / exam`
- `schools.profile_depth`：`priority / foundation`
- `confidence`（routes）：`high / medium / low`

**行级安全（RLS）**：Supabase 启用后，对 `schools / policies / news / knowledge_pages` 设公开只读策略；写操作仅 `service_role` 经 cron/迁移执行。
